# Cursor configuration

This directory is generated by Harness when `--agent cursor` or `--agent all` is selected.

## What's here

| Path | Purpose |
|---|---|
| `.cursor/rules/harness.mdc` | Always-applied rule that explains how to use the installed skill rules. |
| `.cursor/rules/<skill-id>.mdc` | Agent Requested Cursor rule generated from the shared Harness skill source. |
| `.cursor/rules/<skill-id>/` | Supporting reference files for a skill, when the source skill includes them. |

Cursor project rules live in `.cursor/rules/` and use `.mdc` files with frontmatter. Each generated skill rule is intentionally Agent Requested so Cursor can select it from the task context instead of forcing every skill into every prompt.

## Extending

Add new shared skills to the package template source, then reinstall with `--agent cursor` or `--agent all`. Avoid editing generated `.cursor/rules/*` files by hand unless you want project-specific overrides.
