# External Integrations

**Analysis Date:** 2026-07-08

## APIs & External Services

**MCP Server Catalog:**
- GitHub MCP - Adds a GitHub MCP server to generated agent config.
  - SDK/Client: `npx -y @modelcontextprotocol/server-github` (`src/mcps.js`)
  - Auth: `GITHUB_PERSONAL_ACCESS_TOKEN`
- Neon MCP - Adds a Neon database management MCP server to generated agent config.
  - SDK/Client: `npx -y @neondatabase/mcp-server-neon start` (`src/mcps.js`)
  - Auth: `NEON_API_KEY`
- Stripe MCP - Adds a Stripe MCP server with all tools enabled.
  - SDK/Client: `npx -y @stripe/mcp --tools=all` (`src/mcps.js`)
  - Auth: `STRIPE_SECRET_KEY`
- Resend email MCP - Adds an email-sending MCP server.
  - SDK/Client: `npx -y mcp-send-email` (`src/mcps.js`)
  - Auth: `RESEND_API_KEY`

**Remote MCP Servers:**
- Custom HTTPS MCP endpoints - Users can add remote MCP entries with `harness add mcp <name> --url <url> --header KEY=VALUE`.
  - SDK/Client: Native MCP config generated by `src/mcps.js` and `src/commands/sync.js`.
  - Auth: Header placeholders supplied through `--header` and collected by `collectEnvReferences()` in `src/mcps.js`.

**Package Distribution:**
- npm/npx - CLI usage is documented as `npx skogharness@latest` in `README.md`; local global install uses `npm link` in `scripts/install-global.sh`.
  - SDK/Client: npm CLI / npx
  - Auth: Not detected in repository configuration.

**Claude Code Plugin Marketplace:**
- Claude plugin metadata - `.claude-plugin/marketplace.json` exposes `toon-formatter` and `harness-creator` from `templates/.claude/skills/`.
  - SDK/Client: Claude Code plugin marketplace metadata file.
  - Auth: Not detected.

**Agent Native Config Targets:**
- Claude Code - `src/commands/sync.js` writes skills/settings and MCP JSON under `.claude/`, `CLAUDE.md`, and `.mcp.json` in target projects.
  - SDK/Client: Generated `.mcp.json` with `mcpServers` object from `buildMcpServersMap()` in `src/mcps.js`.
  - Auth: MCP env placeholders from `src/mcps.js` or user-provided MCP entries.
- Codex - `src/commands/sync.js` writes Codex skills and managed MCP TOML under `.codex/` and `AGENTS.md` in target projects.
  - SDK/Client: Generated `[mcp_servers.*]` TOML from `buildCodexMcpToml()` in `src/mcps.js`.
  - Auth: MCP env placeholders from `src/mcps.js` or user-provided MCP entries.
